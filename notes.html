Safety

  1. EMP
    • Electromagnetic pluse
    • EMP is when electricity improperly jumps from one place to another in ways it can cause damage.
    • e.g. Lighting and nuclear detonations
  2. ESD
    • Electrostatic Discharge
      • ESD is the EMP of tech person's concern
      • ESD means the passage of static electrical charge from one item to another.
      • ESD taek place when two objects that store different amounts (potential) of static electricity come in contact.
      • ESD can destroy computer's sensitive parts.
      • Avoid ESD by grounding yourself to computer device by having you and computer you touch are at same electrical potential.
  3. ESD straps,bags,and mats
  4. Resistors
    • ESD straps and mats use resistors: devices that stop or resist the flow of electricity - to prevent a static charge from racing through the device.
  5. EMI
    • EMI: Electromagnetic interference is when a magnetic field interfering with electronics.
    • EMI is less dangerous than ESD but can still cause serious damage to parts (HDD).
    • Keep magnets away from computer.
  6. RFI
    • RFI is Radio frequency interferenc, strange noises from speakers.
    • RFI becomes big issue when two devices share same frequencies.(phones, WAP, baby monitors, microwave)

CPU

  1. CPU core components
    • Man in the Box
      • CUP = man in the box
      • EDB: External data bus - connects lightbulb inside the box with lightbulb outside
      • Registers aka worktables: Intel 8088
        • Register store internal commands and data
        • 8088 workstable havae 16 light bulbs each, not in pairs.
        • Each light bulbs is controlled by single switch, man in the box control them.
        • Registers are tiny storage area on CPU, made up of microscopic semiconductor circuits that hold charges.
        • 8088 General purpose registers
          • AX
          • BX
          • CX
          • DX
        • 32bit processor General purpose registers
          • EAX
          • EBX
          • ECX
        • 64bit processor General purpose registers
          • RAX
          • RBX
          • RCX
      • Codebook aka instructions for man in the Box
        • Codebook: one for man in the box, one for you.
        • Coodebook commands = microprocessor's machine language.
        • 01010101 could mean "Add AX to BX"
        • Line of codes = machine language commands, place to EDB one at a time.
        • CPU instruction set = all machine language commands that CPU understand.
    • Clock wire aka CLK: A charge on CLK wire tells CPU (man in the Box) that another piece of information is waiting to be processed.
    • Clock cycle: single voltage charge to CLK wire.
    • CPU requires at least two clock cycles to act on a command, up to hunderds
    • Clock speed: maximum number of clock cycles taht CPU can handle at given period of time.
      • 8088 clock speed: 4.77 MHz or 4.77 million cycles per second.
      • 1 hertzz (1 Hz) = 1 cycle per second
    • System crystal: determine speed at which CPU and rest of PC operate.
      • System crystal is quartz oscillator, similar to wristwatch, soldered to MOBO. it sets the speed.
      • Quartz oscillator sends out electric pulse at certain speed (milliions per second).
      • Every device soldered to motherboard is designed to run at speed of system crystal.
      • 200MHz motherboard's chipset all timed by 200MHz crystal.
    • Clock chip
      • Quartz oscillator's signal goes first to clock chip
      • Clock chip adjust the pulse (usually increase the pluse by multiplying).
    • Overlock CPU
      • telling clock chip to multiply pulse faster than CPU's design speed.

RAM

  1. RAM: made up of groups of semiconductor chips soldered onto small cards.(digital spreadsheet)
  2. DRAM: Dynamic RAM needs both constant electrical charge and periodic refresh of circuits.
  3. MCC: Memory Controller Chip
    • Chip between CPU and RAM.
    • CPU tell which info from RAM it wants to MCC, MCC grab from RAM and put on EDB for CPU.
  4. Address Bus
    • Communicate CPU with MCC for CPU to ask what it want MCC to do in RAM
    • 8088 had 20 wires on address bus
      • Total patterns for 20 wires
      • 1 wire = only ON or OFF
      • 2 wire = 4 combinations 00, 01, 10, 11
      • 20 wire = 1,048,576 = 1MB-ish (technically 1MiB mebibyte)
      • 32bit CPU address bus = 4GB of RAM
      • 64bit CPU address bus = 16EB of RAM

CPU Technology

  1. Clock multipliers
    • Modern CPU: run at some multiple of system clock speed (x15,x20,x30)
    • CPU was only thing doing any work much of time
    • CPU computing process speed up but nothing else.
    • MOBO sets the speed and multiplier automatically(manually overwrite by Overclocking CPU)
  2. 64bit processing
    • 32bit processing = 4GB of RAM via address bus
    • 64bit processing = 16EB of RAM via address bus
    • Primary benefit of 64bit computing = support more than 4GB of memory.
    • x86
      • The term "x86" refers to a family of instruction set architectures based on the Intel 8086, not because it's a 32-bit architecture,
      • but because the names of its successors (80186, 80286, 80386, etc.) all ended in "86
    • x64
    • x86-64
  3. Virtualization support
    • Virtualization support: support for more than one OS at a time.
    • Intel and AMD has built in featuere
  4. Parallel execution
    • Parallel execution: CPU processing multiple commands and parts of commands in parallel.
    • Parallel execution processing stages
      • Pipelining
        • CPU takes at least 4 steps/stages to get command from data bus,calculate, send back out on data bus.
        • Fetch: Get data from EDB
        • Decode: Figure out what type of command needs to be executed
        • Execute: Perform calculation
        • Write: send data back to EDB
      • Pipeline: conveyer belt like fashion,keeps every stage of processor busy on every click of the clock
      • Cache: increased number of stages, up to 20 in some cases.
      • Pipeline stall: stage hits complex command that requires more than one clock cycle, force pipeline to stop.
    • inside CPU
      • CPU composed of multiple chunks of circuitry to handle types of calculations.
      • ALU: arithmetic logic unit (integer unit) handles integer math.
      • FPU: floating point uint handles complex numbers.
      • ALU and FPU operated at execution stage.
      • Pipeline stall happens with FPU. CPU offer multiple pipeline to avoid stall.
    • Cache
      • One click in browser: Windows send many programs (threads and data) to CPU.
      • Threads: these programs breaks down into some number of little pieces called threads.
      • Each thread is a series of instructions designed to do a particular job with the data
    • Wait states
      • CPU is faster than RAM. pipeline stalls are called wait states, where CPU waits for RAM.
      • Static RAM: SRAM is very high speed RAM, built-in to CPU, to reduce wait states.
      • SRAM preloads many instructions as possible
      • SRAM keeps copies of already run instructions and data, using "Cache" feature.
      • SRAM size
        • Early Cache
          • L1 cache = Cache on CPU
          • L2 cache = Cache on Motherboard. Half speed of CPU
          • Connection between CPU, MCC, RAM, L2 Cache
          • Frontside bus = Address bus + EDB(connecting CPU, MCC, and RAM)
          • Backside bus = Connection between CPU and L2 Cache
        • Current Cache
          • Cache on CPU: L1, L2, L3
          • Connection: Integrated Memory Controller
    • Multithreading
      • Multithreading enables CPU to run multiple threads at same time.
      • Multithreading limitations
        • OS and Application must be designed to take advantage of the feature
        • CPU simulates actions of a second processor, but doesn't double the processing power.
  5. Multicore processing
    • dual-core architecture = combine 2 CPUs(or cores) into single chip.
    • Dual-core CPU: 2 execution units, 2 sets of pipelines: share caches and RAM.
  6. IMC integrated memory controller
    • IMC is now in CPU, not in MOBO.
    • IMC optimize the flow of information into and out from CPU.
  7. GPU graphics processing uint
    • integrated GPU into CPU enhances overall performance of computer
  8. Security
    • CPU employ NX bit Technology that enables CPU to protect certain sections of memory.
    • Micrsoft term: DEP Data Execution Prevention, turn on by default in OS.
    • NX bit Technology
      • Intel: XD bit (eXecute Disable)
      • AMD: Enhanced Virus Protection
      • ARM: XN (eXecute Never)

Firmware

  1. Talks
    • Peripherals (kb,mouse) communicate with CPU via physical connection + controller
    • Controller is a chip that connects Peripherals device to CPU, act as an interface.
    • Early motherboard packed with controller chips (memory,keyboard, hard drive,other)
    • Chipsets= combine multiple controller chips.
    • Intel 430VX chipset = northbridge and southbridge
    • Today, controller chips built into CPU (memory controller and dislpay controller)
    • PCH: platform controller hub, named by Intel. AMD and the rest still call it "chipset", even though it is a single chip, not a set.
    • Chipset extends "data bus" to every device (kb,speaker, network, printer, usb) on PC.
  2. Talking to Keyboard
    • Device driver aka codebook: sit on PC's mass storage, loaded into memory as OS starts.
    • OS stores hunderes of device drivers and loads on boot.
    • Without device driver, OS can't run that specific controller (keyboard, mouse, network).
  3. ROM
    • ROM: device derviers berfore the device drivers.
    • ROM nonvolatile and read-only.
    • flash ROM is the one PC motherboard uses.
    • flash ROM + firmware + UEFI
  4. UEFI
    • UEFI is a firmware programming standard that defines how we configure utilities thae every system needs:
      • device drivers
      • boot support
      • system setup
    • UEFI providse the programming thate enables CPU to communicate with other hardware.
    • Correct term for UEFI "device drivers" is services.
  5. BIOS
    • BIOS designed to on Intel 8088, replaced by UEFI.
    • Basic input output system
  6. CMOS
    • every UEFI needs a tiny bit of writable memory,and flash ROM is read-only.
    • Why UEFI needs writable memory?
      • if you want boot-up password, where do you tell UEFI to store that password
      • how will UEFI know how much RAM it needs?
    • Writable memory = tiny bit of speciaized RAM attached to small battery to keep it working when PC is off and unplugged.
    • The memory is call CMOS: complementary metal-oxide semiconductor.
    • CMOS used to be its own chip. Today it is built into "chipset".
    • Information about a hardware (mouse) in CMOS have to match the actual hardware specs.If not,computer won't access the hardware.
    • Why you need to cahnge/update CMOS data is crucial? Read line above.
  7. System Setup Utility
    • How can you access and modify CMOS data?
    • System Setup Utility: comes with UEFI let you modify CMOS data.
    • System Setup Utility = CMOS = sysstem setup = BIOS setup utility = UEFI setup utility = BIOS settings = UEFI firmware setting

  10. BIOS/UEFI Security Settings
    • Boot options
      • 2 hard drives: which drdive to boot the system from?
      • how to boot from anti-virus USB drive instead of cased-hdd?
    • USB permissions
      • Enable USB boot is fun but bad guy can get in easily.
      • Leaving all USB port enable is high risk
      • Law enforcement and such environment will turn off USB ports.
    • Fan
      • Monitor fans and control system fans
    • TPM Trusted Platform Module
      • TPM acts as a secure cryptoprocessor.
      • cryptoprocessor = hardware platform for acceleration of cryptographic functions and secure storage of associated information.
      • TPM is a storage place for very secure keys used by all kinds of software on your PC.
      • Trusted Computing Group publish TPM specifications
      • TCG includes Intel, AMD, IBM, Microsoft, Dell, Lenovo, HP, and others.
      • TPM = small circuit board plugged to MOBO or built-into chipset.
      • TPM usage: in various cryptographic operations
      • Mostly used for "hard disk encryption". Windows BitLocker Drive Encryption feature.
      • Other usage: DRM digital rights management, netework access control, password protection, apllication execution control.
    • Secure Boot
      • Secure Boot includs boot software and software that supports specific, essential components.
      • Secure boot requires intel CPU, UEFI BIOS, and OS designed for it (Windows).
      • Secure Boot to respond to security of boot-up area by making sure device only loads firmware/software trusted by OEM.
      • Trusted software can be achived by "signing the code using digital signature"
      • System runs signature check of each piece of boot software and UEFI drivers, it checks core OS files too.
  11. Exiting and Saving BIOS/UEFI settings
    • Preset/profiles: to save customized settings.
    • Save changes, discard changes, exit saving changes, exit discarding changes
  12. POST
    • system ROM has BIOS but its not the only program.
    • When computer "turn on OR reset", it initiates special program: POST.
    • POST check the system every time the computer boots by sending command to all devices "Check yourself"
    • All devices on system will run built in diagnostic
    • How POST work?
      • POST convey POST info to user either via "beep codes or text messages"
      • Before and during video test: beep code
        • beep code varies by BIOS manufacturers. Comptia term: POST code
        • modern PC beep code
          • Bad or missing video: One Long Beep followed by Two or Three short beeps.
          • Bad or missing RAM: Single beep that repeats indefinitely.
        • Other 3 beep sequences
          • One or Two short beep: end of successful POST indicates all is well.
          • srange noise that repeats: RAM is missing or damaged.
          • beep that aren't related to POST or booting: series of short beeps after running PC for awhile.
      • Text Errors
        • e.g. "Keyboard failure"
        • Inoperative device can disrupt POST and force PC into endless loop. Get POST card.
        • POST card: montor POST and identify which hardware is causing the trouble.
    • Boot Process
      • Power PC
      • Power supply circuitry tests for proper voltage and sends signal down on special wire called "power good wire" to awake CPU.
      • CPU immediatly sends built-in memory address via address bus.
      • CPU's built-in memory address is "first line of POST program on system's ROM". and POST starts.
      • After POST,computer need to find programs on hard drive to start OS.
        • BIOS
          • POST passes control to the last BIOS function: bootstrap loader.
          • Bootstrap loader: lines of BIOS code tacked to the end of POST program, its job is to find OS.
          • Bootstrap loader reads CMOS information for finding OS. (System Setup Utility - Boot Sequence)
          • Boot sector: specific location on storage drives, are configured to boot OS.
          • Bootable device's boot sector contains "special programming" designed to tell system where to locate OS.
          • Boootable disk or system disk: any device with functional OS.
          • Bootstrap loader -locate- good boot sector: pass control to OS and removes itself from memory.
          • Bootstrap loader - can't locate good boot sector: go to next boot device in boot sequence.
        • UEFI
          • POST passes control of boot process to "Boot manager".
          • Boot manager: checks "boot configuration" and loads "OS boot loader" directly.
          • Boot manager no longer need to scan boot sector or anything.
          • UEFI firmware stores "boot manager" and "boot configuration"
        • PXE
          • PXE: preboot execution environment, BIOS feature, enables to boot without any local storage (OS from server over network.)
          • Installing Windows options
  13. Care and Feeding of BIOS/UEFI
    • Default and Optimized Settings
      • Reset options: Load default settings OR OS optimized defautls.

      • CLRTC: dedicated set of wires on MOBO.
      • Turn off PC. Unplug power.

      • Wait 10 seconds, move SHUNT back to default position.
      • Plug and boot PC.

      • Today: MOBO may have "dedicated clear CMOS button" hardwired to MOBO.
    • Losing CMOS RTC Settings
      • CMOS RAM: needs continuous trickle charge to keep internal clock running and remember settings.
      • CMOS battery: CR2032: give CMOS RAM the charge its needs when PC is off.
      • CMOS information lost Errors symptons
        • CMOS configuration mismatch

        • BIOS time and settings Reset
        • No boot device available
        • CMOS battery state low
        • Windows clock reste to January 1st everytime you boot
        • Pulling and inserting cards
        • Touching MOBO
        • Dropping things on MOBO
        • Dirt on MOBO
        • Faulty power supplies
        • Eletrical surges
      • Above symptons require: Battery replacement.
      • 5V soft power on MOBO can provide electricity to keep CMOS charged and secure data while changing CMOS battery.
    • Flashing ROM
      • Reprogram "flash ROM" to update contents inside.
      • Why falsh ROM or update BIOS? to support new technology (support larger drive, support faster RAM, security)
      • Firmware update aka flashing ROM

Power Supplies

  1. Electricity
    • voltage = pressure of electrons in the wire, measured in V volts.
    • current = number of electron moving pass certain point in wire, current = amperage, measured in A amps/amperes.
    • wattage = V * A
    • ohms = resistance to flow of elctrons.
    • conductor = types of matter that allow electrons to move easily
    • insulator = types of matter that inhibit electron movement
    • circuit breaker = heat sensitive or electromagnetically operated electrical switch reated for specified amps.
    • ground wire = provides path of least resistance for electrons to flow back to ground in case of accidental overflow.
    • fuse = has tiny filament designed to break if subjected to too much current.
    • DC: electrons flow in one direction around continuous circuit
    • AC: electrons flow alternates direction, back and forth, in a circuit
    • AC used by power compaines, DC used by electronic devices.
  2. PC Power
    • VAC = volts of alternating current
    • PSU connector to AC outlet: IEC-320 connector with 3 holes.
      • Hot: hot wire carries electrical voltage
      • Neutral: neutral wire carries no voltage but acts like water drain, completing circuit by returning electricity to local source (breaker panel)
      • Ground: ground wire makes it possible for excess electricity to return safely to the ground
    • Testing AC power
      • hot: outputs approx 115V in US
      • neutral: connects to ground ( 0 V output)
      • ground: connects to ground (0 V output)
    • Multimeter aka VOM (volt-ohm meter) aka DMM (digital multimeter): emasure several aspects of electrical current.
    • Multimeter has 2 probes: analog or digital meter, and dial to set type of test.
    • Multimeter test:
      • continuity: determines electrons can flow from one end of wire to other end.If yes, continuity is.
      • resistance:
      • AC voltage(VAC)
      • DC voltage(VDC)
    • AC adapters
      • Computing devices use AC adapter over internal power supply.
      • Black box with two cables: one to wall, one to device.
      • AC adapter: converts AC to DC, just like power supply.
      • 3 things before plugging AC adapter to device
        • voltage
        • amperage
        • polarity
    • Equipment Grounding
      • computer equipment safety starts with proper grounding.
      • Ground wire: functions as an emergency outlet for excess current in case of any short or malfunction of a device.
      • Sag or brownout: voltage drop well below
      • Spike or power surge: voltage shot far above
      • Surge suppressor
        • UL LCC 1449 for 330-V rating
        • Joule: unit of electrical energy. How much energy surge suppressor can handle before it fails, describe in joule.
        • minimum joule rating: 2000 joules
        • surge suppressor can't handle ultimate surge,lightning strike.
      • UPS
        • contains big battery that provides AC power
        • UPS measured in both volt-amps(VA) and watts
        • Watt: (true amount of power they supply in event of power outrage)
        • VA (volt-amps): amount of power UPS could provide if device took power from UPS in perfect way.
        • 2 Types of UPS
          • Online: devices are constantly powered through UPS Battery
          • Standby: devices are connected to UPS only when AC sags belwo 80-90V.
          • Line-interactive, similar to standby: handle moderate AC sags and surge without need to switch battery power.
    • DC power
      • Power Supply Unit
        • 12V: power motors on devices such hard drive, optical drive
        • 5V: otehr onboard electronics
        • 3.3V: other onboard electronics
      • Motherboard Power Connector
        • 20in
        • 24pin
      • Molex connector
        • 5V and 12V: fans and older drives.
        • Chamfers aka notches: to guide installation.
      • Mini Connector
        • few PSU still support mini connector or Berg connector
        • 5V and 12V to peripherals
        • originally use for 3.5 floppy disk drive connector.
      • SATA connector
        • SATA drive: needs 15pin SATA power connector, L shaped.
        • 3.3V pins are not used in any current iteration of SATA drives and reserved for possible future use.
      • ATX
        • Original ATX power supply's physical feature:
          • motherboard power connector: 20pin P1 conector
          • soft power
        • ATX power supply cables
          • minimum 2 cable: each populated with 2 or more Molex or mini connectors for peripheral power.
        • ATX run 5V to motherboard, always "on", even powered down.
        • PC power switch, tells computer it is pressed. BIOS or OS takes over. its call "soft power".
        • Benefits of soft power
          • prevent user from turning off PC before OS shut down.
          • enables PC to use power saving mode (sleep) and wake up
        • Soft power settings
          • Soft power settings inside CMOS setup.
          • BIOS/CMOS utility - Power management - Power button function - Delay 4 seconds.
      • ATX12V 1.3
        • introduced: 4pin motherboard power connector, P4 power connector, provides 12V
        • Any power supply with P4 connector is called "ATX12V"
        • introduced: 6pin auxiliary connector: AUX connector, supply 3.3 and 5V.
        • AMD mobo needs AUX, Intel mobo needs P4.
      • EPS12V
        • SSI: server system infrastructure, non-ATX mobo, power supply EPS12V
        • 24pin mobo power connector: more current than 20pin.
        • EPS12V comes with AUX connector, ATX12V P4 connector, and unique 8piin connector.
        • Rails
          • DC voltage rails: 3.3V , 5V, 12V
          • 12V rails supplies 18amps. not enough for powerful computer. include multiple 12V rails in PSU was solution
          • OCP: over-current protection- will shut down power supply if currnt goes beyond its cap.
          • single-rail system: single OCP circuit monitors all the pathways
          • multi-rrail system: each pathway gets its own OCP circuit.
          • Today: 12V rails pushing 70amps or more.

      • SFX12V: FlexATX motherboard

      • active PFC eliminates any harmonics: which create humming sound.
      • Wattage requirement

        • Graded for efficiency under voluntary standard program: called 80plus.

        • Gold: 90 percent

Mass Storage

  1. Hard Drives
    • HDD: individual disks/platters, read/write heads on actuator arms, controlled by servo motor
    • Each read/write head has bit-sized "transducer" to read/write to each spot on the hard drive.
    • Spindle speed
      • HDD rotate at a set spindle/rotational speed.
      • Spinning platters measured in RPM (revolutions per minute)
      • 5400RPM, 7200RPM,10000RPM,15000RPM
      • 5 degrees of Celsisu may reduce 2 years of HDD life.
    • Form Factors
      • 2.5 inch
      • 3.5 inch
  2. Solid State Drives
    • SSD = combination of semiconductors and transistors,aka use flash memory chips to store data
    • SSD form factors
      • 2.5inch
      • mSATA
        • replaced by M.2
        • M.2 is half the size with better performance
      • M.2
        • M.2 sizes: four-digits or five-digits number. 2242, 2260, 2280, 22110
        • 00 = width in millimeters, 00 or 000 = lenght
        • 2280 = 22mm wide by 80mm long.
    • MLC: multi level cell memory technology SSD, less reliable, less expensive
    • SLC: single level cell memory tehcnology SSD, more reliable, more expensive
    • 3D NAND: form of MLC, stacks cells vertically, provide increased density and capacity.
  3. SSD performace metrics
    • How fast it can read or write long sequence of data stored in same part of drive
    • How fast it can read or write small chunks of data scatterd randomly around the drive
    • How quickly it responds to single request.
  4. Sequential read/write performance
    • throughput = storage device's top speed, rates it can read/write long sequence of data.
    • throughput == megabytes per second (MBps)
      • HDD: 200MBps
      • SATA SSD: 600MBps
      • NVMe SSD: 2500MBps
  5. random read/write perofromance
    • how many times per second a device can read or write small data at random locations.
    • data chunk 4KB: 4K rad, 4K random write, 4K mixed.
    • IOPS: input/output operations per second or MBps.
      • HDD: 150 IOPS
      • NVMe SSD: hundreds of thousands of IOPS
  6. Latency
    • response time, access time, or latency: measurs how quickly it responds to single request.
    • milliseconds(ms) or microseconds(us)
      • HDD: 20ms
      • SSD: 1ms
  7. Connecting Mass Storage
    • Communication beteween CPU and Storage Device requires 2 main items.
      • Standardized physical connection: CPU, drive controlelr, and physical drive.
        • ST-506: 1980 - 1990
        • PATA: 1990 - 2010 (ATA/ATAPI version 1)
        • SATA: 1990 - Today (ATA/ATAPI version 7)
        • SAS: 2005 - Today
        • M.2: 2010 - Today
      • CPU needs to use standardized protocol: special language to speak to storage to read/write data to device.
    • PATA
      • Comptia term: IDE integraded drive electronis drive.
      • IDE refers to any hard drive with built-in controller, all hard drives are IDE drives technically.
      • IDE cables = 40pin ribbon cables for data COMPTIA called it IDE cable.
        • block airflow, pain to insert
        • limited to 18inches
        • can't hot-swap PATA drives
      • IDE cables = 4pin Molex power connector
      • S.M.A.R.T
        • ATA/ATAPI version 3 introduce SMART
        • Self-monitoring,analysis, and reporting technology
        • SMART tracks errors and error conditions within the drive.
        • Infomation is stored in nonvolatile memory on the drive, examined externally with SMART reader software.
    • SATA
      • Point to point connection between SATA device (HDD,SSD,Optical) and SATA controller(HBA: host bus adapter).
      • SATA data cable: 7 connectors vs PATA 40 wires.
      • Cable lenght: twice PATA , instead 18 inches, SATA is 40 inches (1 meter).
      • Maximum drives: no limit, Snap in SATA HBA into expansion slot for more drive.
      • Data throughput: 30x PATA,
        • 1.5Gbps = max throughput 150MBps
        • 3Gbps = max throughput 300MBps
        • 6Gbps = max throughput 600MBps
        • 1.5Gbps = 192MBps. Encoding scheme used on STA drives take 20% of transferred byes as overhead.
        • SATA utilizes the 8b/10b encoding scheme, where 8 bits of data are transmitted as 10 bits, adding overhead for clock recovery and DC balancing
      • SATA first to support hot swapping
    • SATA varity
      • SATA 1.0: 1.5Gbps/150MBps
      • SATA 2.0: 3Gbps/300MBps
      • SATA 3.0: 6Gbps/600MBps
      • SATA 3.2: 16Gbps/2000MBps
    • SATAe
      • SATAe aka SATA Express aka SATA 3.2: ties drive directly into PCIe bus on motherboard.
      • SATAe drops both "SATA link" and "transport layer", full perofromance of PCIe, no overhead.
      • SATAe has unique connectors
      • Motherboard with SATAe support is required.
    • eSATA and Other External Drives
      • eSATA aka External SATA: extend SATA bus to external devices.
      • eSATA cable: shielded cable in 2 meters lenght. hot swappable.
      • eSATA is gone when USB3.0 appear.
      • Current external drives use: USB (3.0, 3.1, 3.2, C)
  8. Refining Mass Storage Communication
    • AHCI
      • Advanced Host Controller Interface, an efficient way to work with SATA HBA, support by Windows.
      • AHCI unlocks SATA's advanced features:
        • NCQ native command queuing
        • Hot Swapping
      • NCQ is disk optimization feature for SATA drives.
        • NCA take SATA interface advantage and achieve faster read/write speed than PATA.
      • SATA Hot Swapping
        • SATA support hot swapping
        • Motherboard and OS have to support too.
      • AHCI mode is enabled at CMOS level, needs to be enabled before OS installation,if not "blue screen"
      • AHCI disabled: SATA drive won't appear in Windows.
    • NVMe
      • Non volatile memory express: supports communication connection between OS and SSD directly thru PCIe bus lane.
      • Direct connection reduces latency and take full advantage of fast speed SSD.
      • NVMe formats
        • add-on expansion card using PCIe slot
        • 2.5 inch drive works with SATAe connector
        • M.2
  9. Protecting Data with RAID
    • SCSI
      • Personal Computer Market: SATA connected HDD and SSD
      • Server market: SCSI - small computer sytem interface
      • SCSI cable: based on version, used various ribbon cable.
      • SAS: Serial Attached SCSI hard drives: fast and robust storage for server and storage arrays.
      • SAS4 speed: 22.5Gbps
      • SAS controller: support SATA drives.
      • SAS connector type: over a dozen different connector, looks like biger SATA connector
    • Disk Mirroring
      • data on each drive are identical.
      • Safe but slow
      • Both drives: assigned same drive letter.
    • Duplexing
      • Same as mirroring but use separate controller for each drive.
      • exactly two (redundant) drives
      • faster and safer
      • Both drives: assigned same drive letter.
    • Disk Striping (without parity)
      • Spread data among multiple (minimum 2) drives.
      • No redundancy. e.g. 50% of file in Disk1, 50% of file in Disk2
      • Fast but not safe, similar to one-drive, one-controller setup.
      • Two or more (non-redundant) drives
      • Both drives: assigned same drive letter
    • Disk Striping with parity
      • Protect data by adding extra information: aka parity data.
      • Parity data: can be used to rebuild data if one drives fail.
      • Minimum 3 drives
      • Disk striping with parity= combination of disk mirroring and disk striping.
      • Fast and Safe. Majority of nework severs use it.
    • JBOD
      • Just a bunch of disks = multiple independent disks of various sizes.
    • RAID
      • RAID 0
        • Disk striping
        • 2 drives, no redundancy
      • RAID 1
        • Disk mirroring/duplexing
        • 2 drives or any even number of drives
        • Ultimate Safety, data is duplicated
      • RAID 5
        • Disk striping with distributed parity
        • distribute "data and parity information" evenly across all drives.
        • Fastest way for data redundancy
        • 3 drives, 1 drive's space is used for parity.
      • RAID 6
        • Disk striping with extra parity
        • RAID6 is RAID5 with extra parity information.
        • 4 drives, can lose up to 2 drives at same time.
      • RAID 10
        • Nested, striped mirrors aka RAID 1+0 aka "stripe of mirrors"
        • 4 drives, pair of drives configured as mirror, same for other pair = pair of RAID1 mirroring.
        • stripe across two mirrored pairs.
        • Speed of striping, reliability of mirroring
      • RAID 0+1
        • Nested, mirrored stripes
        • 4 drives: 2 RAID0 striped arrays, mirror 2 arrays to each other.
    • Implementing RAID
      • Software RAID
        • Software: price over performance
        • No special controller needed: SATA controller can make Software RAID array.
        • Disk Management in Windows Server - RAID 0,1,5, works with PATA and SATA
        • Windows 10 and 11: RAID 0 and 1.
      • hardware RAID
        • Hardware: speed + redundancy
        • intelligent controller: handle RAID functions,controllers have chips with own processor and memory.
        • Hot swapping - ability to replace bad drive without interrupting OS.
        • Hardware RAID: invisible to OS
        • RAID has special configuration utility in flash ROM - access after CMOS but before OS loads.
    • Dedicated RAID boxes

      • HDD or SSD: needs SATA connection
